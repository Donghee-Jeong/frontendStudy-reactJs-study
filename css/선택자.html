선택자

복합 선택자

  1. 일치생성자 EF (E와 F를 동시에 만족하는 요소)
    id가 orange인 span태그
  2. 자식 선택자 E>F (E의 자식요소 F)
    div의 자식요소 span
  3. 후손(하위) 선택자 E F (E의 후손 요소 F, '띄어쓰기'가 선택자의 기호로 사용됨)
    div1의 후손이고, div2의 자식인 span태그
  4. 인접 형제 선택자 E+F (E의 다음 형제 요소 F 하나만 선택)
    형제1
     형제1의 다음 형제 요소인 형제2 (br태그도 있으면 안됨)
    형제1의 다다음 형제 요소인 형제2
  5. 일반 형제 선택자 E ~ F (E의 다음 형제 요소 F 모두 선택)
    • 리스트1
    • 리스트2
    • 리스트3
      • span1
    • 리스트4

가상 클래스 선택자

  1. nth-child(n)
    E:nth-child(n) E가 형제 요소 중 n번째 요소라면 선택(n은 1부터)
    1
    2
    3
    4
    nth-child(2n) 으로 짝수번째 요소들만 선택할 수 있다.
    1
    2
    3
    4
    nth-child(n+3) 은 3번째 요소부터 이후 요소들을 선택.
    1
    2
    3
    4
    E:nth-child(n)은 E가 형제 요소들 중에 n번째 요소가 아니면 적용되지 않는다.
  2. 위의 문제 해결을 위한 nth-of-type(n)
    E:nth-of-type(n) E의 타입과 동일한 타입인 형제 요소 중 E가 n번째 요소라면 선택
    div딸기

    p사과

    p망고

    E의 타입이란 클래스든 id이든 그것의 요소의 타입(태그이름)을 말하는 것으로,
    같은 태그 중에 E가 n번째인지 확인하는 것
    • 오렌지
    • 딸기
    • 망고
    • 사과
  3. 부정 선택자
    E:not(S) S가 아닌 E 선택
    • 딸기
    • 사과
    • 오렌지
    • 망고

속성 선택자

  1. ATTR
    [attr] 속성 attr을 포함한 요소 선택
    id가 있는 div
    id가 없는 div
  2. ATTR=VALUE
    [attr=value] 속성 attr을 포함하며 속성 값이 value인 요소
  3. ATTR^=VALUE
    [attr^value] 속성 attr을 포함하며 속성 값이 value로 시작하는 요소
    class asd
    class akr
    class bdd
  4. ATTR$=Value
    [attr$=value] 속성 attr을 포함하며 속성 값이 value로 끝나는 요소
    class kk-k
    class ak-k
    class bd-d
